# 5.3 Delete album API

## Summary

Backend DELETE `/api/albums/:id` with album token and creator check. List media, delete each from Storage (storagePath, previewPath, thumbnailPath), delete media docs, delete album doc. Return 204. 403 if not creator.

## Approach

- **Backend**: Firestore: `listMedia(albumId)`, `deleteDoc(collection, id)`, delete media subcollection docs. Storage: `deleteFile(path)`. DELETE handler: require creator, list media, for each delete 3 storage paths then media doc, then delete album doc, 204.
- **Security**: Creator-only; 403 for non-creator.

## Files to add or change

- `backend/src/lib/firestore.ts` — listMedia(albumId), deleteDoc(collection, id), deleteMediaDoc(albumId, mediaId) or equivalent
- `backend/src/lib/storage.ts` — deleteFile(path)
- `backend/src/routes/albums.ts` — DELETE `/:id`

## Acceptance criteria

- [x] Creator gets 204 and album + media are removed from Firestore and Storage
- [x] Non-creator gets 403

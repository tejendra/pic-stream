# Feature 8.2–8.5 — Signed URLs, delete media, grid, video polling

## Summary

Implement GET signed download URLs, DELETE media (Firestore + Storage), and the album media grid with modal viewer, download, delete, and 10s polling for video processing.

## Approach

- **Backend**: `readMediaDoc` for single media reads. GET `.../media/:mediaId/url` validates `type`, resolves `storagePath` / `previewPath` / `thumbnailPath`, checks path is under album, returns GCS v4 read URL with 1h expiry. DELETE removes all present Storage paths then the media doc.
- **Frontend**: `listMedia`, `getMediaSignedUrl`, `deleteMedia` on the client. Album page loads media with the album, builds grid URLs (image → original; video → thumbnail, else preview when available, else “Processing…”), modal uses preview or original for video, download uses API → signed URL → blob → `<a download>`, delete confirms then DELETE and refetch. Poll list every 10s while any video has `previewPath === null`.

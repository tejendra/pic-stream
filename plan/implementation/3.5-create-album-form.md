# 3.5 Create album form (frontend) – Implementation plan

## Summary

Frontend create-album page with Material UI: form (name, delete-on date, creator name), POST `/api/albums`, show seed with Copy and Download, store token/albumId in localStorage under `album_${albumId}`, redirect to `/album/${albumId}`.

## Approach

- **Frontend**: Use Material UI (TextField, Button, Box, Typography, etc.). Create page at route `/create` (or `/`) with form; on submit call `createAlbum`, then show "Save this seed" with Copy (navigator.clipboard.writeText) and Download (Blob + anchor download). Save `{ token, albumId }` to `localStorage.setItem('album_${albumId}', JSON.stringify(...))`. Redirect via `navigate('/album/' + albumId)`.
- **API client**: Add `createAlbum(request: CreateAlbumRequest): Promise<CreateAlbumResponse>` that POSTs to `apiUrl('/api/albums')` with JSON body.
- **Routing**: Add route `/create` for create-album page; keep or update Home to link to it.
- **Theme**: Wrap app in MUI ThemeProvider (optional but recommended).

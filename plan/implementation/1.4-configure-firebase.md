# 1.4 Configure Firebase – Implementation plan

## Summary

Configure backend to use Firebase Admin SDK for Firestore and Storage (default credentials on Cloud Run, `GOOGLE_APPLICATION_CREDENTIALS` for local dev). Frontend stays Firebase-free and uses `VITE_API_URL` for the API base. No runtime errors when Firebase env is missing.

## Approach

- **Backend:** Add `firebase-admin`. Initialize only when `GOOGLE_CLOUD_PROJECT` (and optionally `GOOGLE_APPLICATION_CREDENTIALS` for local) are set. Lazy init in lib modules so the app starts even without Firebase config. Expose Firestore read/write and Storage signed URL generation via `lib/firestore.ts` and `lib/storage.ts`.
- **Frontend:** No Firebase SDK. Add `VITE_API_URL` to env example and a small API client helper that uses it (or document for future use). Frontend already uses proxy in dev; in prod it will use `VITE_API_URL`.
- **Security:** No secrets in repo. Backend uses application default credentials (env or Cloud Run).

## Files to add or change

| Path | Action |
|------|--------|
| `backend/package.json` | Add firebase-admin |
| `backend/src/config.ts` | Add firebase project + bucket, isConfigured flag |
| `backend/src/lib/firestore.ts` | Lazy init Firestore; getDb(); read/write helpers |
| `backend/src/lib/storage.ts` | Lazy init Storage; getBucket(); getSignedUploadUrl(path) |
| `backend/.env-local-example` | Add GOOGLE_CLOUD_PROJECT, FIREBASE_STORAGE_BUCKET, GOOGLE_APPLICATION_CREDENTIALS (optional) |
| `frontend/.env.example` or env doc | Add VITE_API_URL (frontend uses for API base) |
| `README.md` | Document new env vars (implemented only) |

## Acceptance criteria

- [x] Backend can read/write Firestore and generate signed Storage URLs
- [x] Frontend has no Firebase SDK and only uses `VITE_API_URL`
- [x] No runtime errors from missing Firebase config in backend
- [x] Firebase Hosting configured for SPA (`firebase.json`: `frontend/dist`, SPA rewrite)
- [x] Firestore and Storage rules in repo; deployable via `firebase deploy`
- [x] Backend deployable to Cloud Run (Dockerfile, build via Cloud Build, docs)
- [x] Backend config defaults to project from `.firebaserc` when env vars unset
- [x] Deploy order documented: deploy backend → set `VITE_API_URL` to Cloud Run URL → build frontend → deploy hosting
- [x] Docs: how Firebase is used, how to configure and deploy Cloud Run

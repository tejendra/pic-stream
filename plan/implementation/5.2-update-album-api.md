# 5.2 Update album API

## Summary

Backend PATCH `/api/albums/:id` with album token and creator check. Body `{ deleteOn?: string, name?: string }`. Update Firestore; return 200 with updated album. 403 if not creator.

## Approach

- **Backend**: Add `requireCreator` middleware (403 if `!req.albumToken?.creator`). Add `updateDoc(collection, id, partial)` in firestore. PATCH handler: validate body, update album doc with only provided fields, return 200 with same shape as GET (name, deleteOn, createdBy, isCreator).
- **Security**: Creator-only; non-creator gets 403.
- **Types**: Add `UpdateAlbumRequest`, `UpdateAlbumResponse` (same as GetAlbumResponse).

## Files to add or change

- `shared/types.ts` — UpdateAlbumRequest, UpdateAlbumResponse
- `backend/src/lib/firestore.ts` — updateDoc
- `backend/src/middleware/auth.ts` — requireCreator (or new middleware)
- `backend/src/routes/albums.ts` — PATCH `/:id`

## Acceptance criteria

- [x] Creator can PATCH and get 200 with updated album
- [x] Non-creator gets 403
- [x] Firestore doc is updated

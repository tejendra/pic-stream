# 3.3 Implement open album API – Implementation plan

## Summary

Implement POST `/api/albums/open` with body `{ seed: string }`: query all albums, find the one where `bcrypt.compare(seed, album.seedHash)` is true, return 200 with `{ token }` (JWT: albumId, exp 24h, creator: false) or 401 if not found.

## Approach

- **Backend**: In `backend/src/routes/albums.ts`, add POST `/open` handler. Validate body has `seed` (string). Get Firestore, fetch all docs from `albums` collection. For each album, `bcrypt.compare(seed, album.seedHash)`; on first match, sign JWT with `{ albumId, exp: 24h from now, creator: false }`, return 200 `{ token }`. If no match, return 401. Reuse TOKEN_EXPIRY_HOURS and config.jwtSecret. Do not log the seed.
- **Firestore**: No index needed for “open by seed” (we scan albums and compare server-side). Add a small helper in firestore.ts to list all album docs if desired, or query collection in the route.
- **Security**: Only return token on correct seed; wrong seed gets 401. Token has creator: false so middleware (3.4) can restrict creator-only actions.

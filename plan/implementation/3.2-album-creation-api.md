# 3.2 Implement album creation API – Implementation plan

## Summary

Implement POST `/api/albums` with body `{ name, deleteOn, createdBy }`: generate seed via `generateSeed()`, hash with bcrypt, create Firestore album doc, return 201 with `{ albumId, seed, token }` (JWT, 24h exp, creator: true). Never log the seed.

## Approach

- **Backend**: Add `bcrypt` and `jsonwebtoken` (and types). Add `config.jwtSecret` from `process.env.JWT_SECRET`. Create `backend/src/routes/albums.ts` with POST handler: validate body (CreateAlbumRequest), call `generateSeed()`, `bcrypt.hash(seed, 10)`, generate album id (Firestore-style or nanoid), write doc to `albums` with id, name, seedHash, deleteOn, createdAt (ISO), createdBy. Sign JWT with payload `{ albumId, exp: 24h from now, creator: true }`. Return 201 and `{ albumId, seed, token }`. Do not log the seed or the response body.
- **Security**: Store only seedHash in Firestore. JWT_SECRET required for signing; reject request or return 500 if missing. No seed in logs.

# 2.2 Document Storage path layout and path helper – Implementation plan

## Summary

Document Firebase Storage path layout and implement a path helper that validates and builds paths under `albums/{albumId}/` only, rejecting `..`, enforcing 200-character limit, and allowing only `[a-zA-Z0-9._-]` in filenames/segments.

## Approach

- **Backend**: Add `backend/src/lib/paths.ts` with:
  - Validation: reject any segment containing `..`; allow only `[a-zA-Z0-9._-]` in segment values (albumId, uniqueId, sanitized filename); enforce total path length ≤ 200.
  - Builder: `buildStoragePath(albumId, type, uniqueId, filename?)` where `type` is `'originals' | 'previews' | 'thumbnails'`. For thumbnails, path is `albums/{albumId}/thumbnails/{uniqueId}.jpg` (no filename). For originals/previews, path is `albums/{albumId}/{type}/{uniqueId}_{sanitizedFilename}`. Returns path string or `null` if invalid.
  - Filename sanitization: take basename (strip path), allow only `[a-zA-Z0-9._-]`; reject (return null) if segment contains `..` or disallowed characters.
- **Docs**: Add `docs/storage-paths.md` documenting the three path patterns and that all paths lie under `albums/{albumId}/`.
- **Security**: Path helper ensures no path traversal (`..`), controlled charset, and length limit so Storage paths stay within the album prefix.

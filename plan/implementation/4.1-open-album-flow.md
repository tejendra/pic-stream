# 4.1 Open album flow (frontend) – Implementation plan

## Summary

Frontend "Open album" page: single text input for 5-word seed, POST `/api/albums/open` with `{ seed }`. On 200 store token (and albumId from JWT payload) in localStorage under `album_${albumId}`, redirect to `/album/${albumId}`. On 401 show "Invalid seed".

## Approach

- **API client**: Add `openAlbum(seed: string)` that POSTs to `/api/albums/open`, returns `{ token }`. Decode JWT payload on client to get `albumId` for storage key and redirect (no backend change).
- **OpenAlbumPage**: MUI page at `/open` with TextField (multiline or single line) for seed, Submit button. On submit call `openAlbum(seed)`; on success store `localStorage.setItem('album_'+albumId, JSON.stringify({ token, albumId }))`, then `navigate('/album/'+albumId)`. On 401 (or non-200) set error state to "Invalid seed".
- **Routing**: Add route `/open`, add "Open album" link on Home.

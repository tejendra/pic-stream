# 2.3 Add shared TypeScript types – Implementation plan

## Summary

Add TypeScript interfaces for Album, Media, and API request/response shapes in a single **shared** package. Backend and frontend depend on `shared` so payloads stay consistent with no duplication.

## Approach

- **Shared package**: Added `shared/` at repo root with `types.ts` (Album, Media, and all request/response interfaces) and `package.json` (`"shared": "file:../shared"` consumed by backend and frontend). Single source of truth; no build step (TypeScript and Vite resolve the .ts file).
- **Backend**: Dependency on `shared`; `backend/src/index.ts` imports from `'shared'`. Remove `backend/src/types.ts`.
- **Frontend**: Dependency on `shared`; `frontend/src/api/client.ts` imports from `'shared'` and re-exports types for API usage. Remove `frontend/src/types.ts`.

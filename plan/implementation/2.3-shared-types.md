# 2.3 Add shared TypeScript types – Implementation plan

## Summary

Add TypeScript interfaces for Album, Media, and API request/response shapes in `backend/src/types.ts` and matching interfaces in `frontend/src/types.ts`. Ensure backend and frontend import from their local types file so payloads stay consistent.

## Approach

- **Backend**: Create `backend/src/types.ts` with Album (Firestore), Media (Firestore), and request/response types for create album, open album, GET album, prepare upload, finalize upload, and media list. No runtime code in types file.
- **Frontend**: Create `frontend/src/types.ts` with the same interface names and field types (copy structure) so request/response shapes are identical. Frontend API client or components can import these for typed requests/responses.
- **Wire imports**: Add at least one import from `./types.js` in backend (e.g. in `index.ts` or a route) and from `./types` in frontend (e.g. in `api/client.ts`) so the acceptance criterion "Backend code imports from backend/src/types.ts" and "Frontend code imports from frontend/src/types.ts" are satisfied. Future routes and API calls will use these types as they are implemented.

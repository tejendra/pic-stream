# 5.1 Get album details API

## Summary

Backend GET `/api/albums/:id` with album token auth: return 200 with `{ name, deleteOn, createdBy, isCreator }`; return 403 when token.albumId does not match `:id`.

## Approach

- **Backend**: Handler already exists. (1) Change auth middleware to return **403** (not 401) when `token.albumId !== :id`.
- **Security**: 403 for forbidden (valid token, wrong album) keeps 401 for missing/invalid/expired token.
- **Types**: Make `GetAlbumResponse.id` optional so frontend can use albumId from route when id is omitted.

## Files to add or change

- `backend/src/middleware/auth.ts` — return 403 on albumId mismatch
- `backend/src/routes/albums.ts` — response body: only name, deleteOn, createdBy, isCreator
- `shared/types.ts` — GetAlbumResponse: id optional

## Acceptance criteria

- [x] Authenticated request with matching albumId returns 200 and the four fields
- [x] Mismatch returns 403

# 4.2 Recent albums list (frontend) – Implementation plan

## Summary

Store recent albums in localStorage as `recentAlbums` (array of `{ albumId, name }`). Landing page shows the list; on click check token from `album_${albumId}` — if missing or expired show "Re-enter seed" link to /open, else navigate to album.

## Approach

- **Storage**: Key `recentAlbums`, value JSON array of `{ albumId, name }`. Helper to add/update (prepend, dedupe by albumId, cap at e.g. 20) and to read list.
- **When to add**: CreateAlbumPage after success (we have name). OpenAlbumPage after success: call GET /api/albums/:id with token to get name, then add to recentAlbums.
- **API client**: Add `getAlbum(albumId, token)` for GET with Bearer token. Add `isTokenExpired(token)` (decode JWT payload, compare exp to now).
- **Home**: On load read recentAlbums, render MUI List. On item click: read `album_${albumId}` from localStorage; if missing or token expired, show "Re-enter seed" with Link to /open; else navigate to /album/:id.

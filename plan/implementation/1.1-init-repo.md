# 1.1 Initialize repo – Implementation plan

## Summary

Initialize repo with top-level `frontend/` and `backend/` directories and root `package.json` with npm workspaces.

## Approach

- **Scope:** Repo structure only. No app code; later work items will scaffold frontend and backend.
- **Root:** Add `package.json` with `"workspaces": ["frontend", "backend"]`.
- **Directories:** Create `frontend/` and `backend/`. Each needs a minimal `package.json` so npm recognizes them as workspace packages (required by npm workspaces).

## Files to add or change

| Path | Action |
|------|--------|
| `package.json` | Create (root, workspaces only) |
| `frontend/package.json` | Create (minimal, name + private) |
| `backend/package.json` | Create (minimal, name + private) |

## Acceptance criteria

- [ ] Repo root contains exactly `frontend/` and `backend/` directories
- [ ] Root `package.json` exists with `"workspaces": ["frontend", "backend"]`

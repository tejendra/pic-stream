# 5.4 Album management UI

## Summary

Frontend: Album page loads GET `/api/albums/:id`. If `response.isCreator` show "Edit delete date" and "Delete album". Edit: modal with date input, PATCH `{ deleteOn }`. Delete: confirm dialog, then DELETE, redirect to `/`, clear localStorage (recentAlbums and `album_${albumId}`).

## Approach

- **Frontend**: AlbumPage gets id from params, token from localStorage `album_${id}`. If no token or expired, redirect to `/`. getAlbum(id, token). If isCreator: show Edit delete date (modal with date input), Delete album (confirm "Delete entire album? This cannot be undone."). Edit: on save patchAlbum(id, token, { deleteOn }), refresh album state. Delete: on confirm deleteAlbum(id, token), removeFromRecentAlbums(id), localStorage.removeItem(album_${id}), navigate to `/`.
- **API client**: patchAlbum(albumId, token, body), deleteAlbum(albumId, token).
- **recentAlbums**: add removeFromRecentAlbums(albumId).

## Files to add or change

- `frontend/src/lib/recentAlbums.ts` — removeFromRecentAlbums(albumId)
- `frontend/src/api/client.ts` — patchAlbum, deleteAlbum
- `frontend/src/pages/AlbumPage.tsx` — load album, creator controls, edit modal, delete confirm

## Acceptance criteria

- [x] Creator sees both controls (Edit delete date, Delete album)
- [x] Edit updates date via PATCH
- [x] Delete shows confirmation and then redirects and clears localStorage for this album

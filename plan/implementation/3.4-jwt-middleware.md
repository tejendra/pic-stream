# 3.4 Implement JWT middleware – Implementation plan

## Summary

Implement middleware that verifies `Authorization: Bearer <token>`, decodes with JWT_SECRET, ensures `token.albumId` matches route param `:id` or `:albumId`, attaches payload to `req.albumToken`, and returns 401 otherwise.

## Approach

- **Backend**: Add `backend/src/middleware/auth.ts`. Parse `Authorization` header (Bearer scheme). Verify and decode JWT with `config.jwtSecret`. Read `albumId` from route via `req.params.id ?? req.params.albumId`; if token.albumId does not match, return 401. On success set `req.albumToken` to decoded payload `{ albumId, exp, creator? }`. Return 401 for missing header, invalid token, or albumId mismatch.
- **Types**: Extend Express `Request` so `req.albumToken` is typed (e.g. `backend/src/express.d.ts` or inline in middleware).
- **Apply**: Use middleware on routes that have `:id` or `:albumId`. Add at least one such route (e.g. GET `/api/albums/:id`) so the middleware can be exercised and AC verified.
